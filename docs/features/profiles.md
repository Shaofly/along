# 个人主页与资料媒体

个人主页是一张持续生长的生活扉页：先帮助来访者辨认“这是谁”，再进入对方有权看见的个人表达和共同经历。它不是公开简历、数据榜单或陌生人发现页。

## 实施状态

| 能力 | 状态 | 当前边界 |
| --- | --- | --- |
| 连续生活扉页 | `已实现` | 桌面封面向背景长距离渐隐，头像、身份、操作和连续内容流保持同一表面 |
| 头像、封面与焦点 | `已实现` | 使用受保护媒体、五种主题回退和独立裁切焦点 |
| 资料编辑 | `已实现` | 桌面弹窗、手机独立任务页；基本资料、个人信息、资料隐私、头像、封面和主题一起保存 |
| 个人信息隐私 | `已实现` | 性别、规范化现居地、手机号、独立联系邮箱和学校共用“可访问我的人 / 指定朋友 / 私密”权限 |
| 内容筛选与分页 | `已实现` | 全部、个人动态、共同经历；本人另有私密动态筛选和草稿入口 |
| 登录邮箱与用户编号 | `已实现` | 只向本人显示，邮箱第一版只读 |
| 最近来过、个人归档 | `未来设想` | 当前没有完整数据模型，不显示占位入口 |

## 单一核心任务

来访者应在一屏内确认主页主人、理解双方关系范围，并开始浏览自己有权看见的记录。本人额外能够发布动态和整理资料，但编辑入口不能压过内容本身。

## 身份与可见范围

个人主页不公开索引。只有以下关系能够打开：

- 本人；
- 直接朋友；
- 与主页主人至少共享一个当前活跃圈子的成员。

不同关系看到的资料不同：

| 访问者 | 头像 | 封面 | 昵称、真名、简介 | 可选个人信息 | 个人动态 | 共同经历 | 登录邮箱与用户编号 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 本人 | 是 | 是 | 是 | 始终可见 | 全部合法内容 | 有原始权限的内容 | 是 |
| 直接朋友 | 是 | 是 | 是 | 按资料主人的可见范围 | 按原动态权限 | 有原始权限的内容 | 否 |
| 同圈非好友 | 是 | 否，显示主题回退 | 是 | 仅“全部”模式 | 否 | 仅共同圈子中有原始权限的内容 | 否 |
| 无合法关系 | 否 | 否 | 否 | 否 | 否 | 否 | 否 |

头像承担当前关系中的人物识别，可被本人、直接朋友和当前活跃共同圈子成员读取。封面更接近个人生活表达，只向本人和直接朋友开放。同圈非好友即使知道封面媒体编号，也不能通过媒体接口读取。

历史圈子档案只解释冻结时的内容与成员，不授予持续查看当前头像或主页的权限。历史内容中的人物视觉回退为姓名首字；若用户从其他当前关系重新获得权限，页面的普通当前关系查询可以独立显示头像。

## 资料模型

- 真名必填，昵称可选。主页大标题使用真名，小型身份说明使用昵称；本人没有昵称时回退为“我的个人空间”，其他访问关系回退为关系说明。
- 简介最多 160 个字。
- 性别、现居地、手机号、联系邮箱和学校均可选，未填写的字段不输出。现居地在数据库中仍保存为一个文本值，不拆成省、市等固定列；编辑器根据“国内 / 海外”提供两级输入，国内统一为“省级地区 · 城市或地区”，海外统一为“国家或地区 · 城市或地区”，后一项可以留空。这样既能稳定展示“安徽 · 合肥”“安徽”“韩国 · 首尔”，也不会把海外地址强塞进省市模型。
- 联系邮箱与登录邮箱是两个独立字段。用户首次编辑时联系邮箱默认带入登录邮箱，保存后独立维护；清空联系邮箱不会修改登录账号，登录邮箱也不会因资料权限向他人输出。
- 上述可选个人信息共用一个可见范围：界面中的“可访问我的人”（`all`）表示所有本来就有权打开主页的人，`selected` 只允许当前仍是直接朋友且被明确选中的用户，`private` 只允许本人。选择“指定朋友”但没有选中任何人时，保存会在服务端归一为`private`，不能持久化一个没有查看者的`selected`范围；资料原本已经私密时，这一操作不改变此前记住的有效共享配置。服务端直接裁剪字段，不能只在浏览器隐藏。
- 隐私保护快捷按钮始终在原地直接切换：开启时把当前范围临时切换为 `private` 并记住上一个 `all` 或 `selected`，关闭时恢复上次范围；此前没有共享范围时进入 `all`。这里的“全部”仍只覆盖原本有权访问主页的人。快捷按钮与资料编辑器共享同一份客户端状态，接口成功后刷新服务端资料；切换后立即打开编辑器不得显示旧范围。
- 登录邮箱和稳定的 `user.id` 只在本人主页的折叠账号信息中显示；认证提供方的 `account.account_id` 不属于用户界面账号编号，禁止暴露。
- `user.image` 暂时作为旧账号头像回退。新头像和封面使用正式媒体关联，不能把受保护媒体 URL 写回普通字符串字段。
- 头像和封面分别保存水平、垂直焦点。相同图片在手机、平板和桌面采用不同容器比例时，通过焦点计算 `object-position`，不生成设备专属裁切副本。
- 没有封面时从鼠尾草、柔粉、雾蓝、暖杏、深墨五种克制主题中选择一种纯色回退；不使用任意颜色选择器、紫色渐变、装饰光球或玻璃效果。
- 当前桌面主页、手机主页和编辑器封面预览统一使用固定`16:7`取景框；原图与焦点继续保留，为以后按滚动逐步展示更多原图留出空间。渐隐属于主页展示层，不计入另一个裁切比例。

## 页面信息架构

```text
个人扉页
├── 封面或主题回退
├── 头像 / 昵称说明 / 真名
├── 简介 / 开始记录时间 / 有权限的个人信息
├── 本人操作：发布动态 / 编辑资料
├── 本人账号信息：登录邮箱 / 用户编号
└── 本人快捷操作：草稿 / 隐私保护

内容区
├── 全部
├── 个人动态
├── 共同经历
└── 本人：私密动态
```

“私密动态”只在本人内容筛选中出现，其他访问者仍只看到三个公共栏目。草稿属于发布准备，不混入已发布动态流。最近来过和个人归档在真实模型完成前不显示禁用项或虚构数据。

## 内容规则

- **全部**：混合展示当前访问者有权查看的个人动态和共同经历。
- **个人动态**：主页主人发布到个人空间、且当前访问者有权查看的内容。
- **共同经历**：主页主人当前仍是对应圈子活跃成员、被动态明确列为参与者，并且当前访问者同时拥有原动态权限的圈子内容。
- **私密动态**：只向本人展示其 `private` 个人动态。
- 圈子内容仍以原圈子动态为唯一正式版本；主页不复制正文、媒体或权限。
- 成员退出圈子后，该圈子的共同经历从个人主页移除；冻结历史从历史圈子档案进入。
- 内容筛选在服务端执行，不能先获取一批混合数据再在浏览器裁切。
- 内容按 `created_at` 与 `id` 组成的稳定游标分页。首屏不显示截断数组长度冒充“动态总数”；没有真实计数查询时不展示总数。

## 桌面与平板

- 页面使用连续内容背景，不把资料拆成卡片墙。
- 封面使用全宽固定`16:7`取景框，从页面顶端延伸到全站共享的半透明页眉之后；上部保持真实照片，下部通过多段遮罩逐渐过渡为页面背景，头像和身份锚定在取景框约四分之三处，渐隐延续到开始记录时间和个人信息行之后，不保留明显底边。
- 头像位于照片与背景的交界，真名位于头像右侧；简介从头像左边缘开始占据完整身份宽度并居中。
- 真名使用系统中文无衬线字体的中等字重和舒展字距，不为生成式参考图引入无法确认来源的大体积字体文件。
- 发布和编辑按钮使用个人页参考设计中的鲜明草绿与蜜桃色，保持同高、近似等宽；图标和文字在同一行，不使用接近黑色的僵硬主按钮或空白描边次按钮。
- 动态主栏保持约 680 至 720 像素并与身份文字对齐；大屏不无限拉宽正文。
- 桌面页眉复用其他一级页面的紧凑半透明表面：左侧显示品牌标识和一级分段导航，右侧依次显示草稿、通知以及无外层胶囊底色的“头像 / 最多四字昵称 / 展开箭头”账号入口；没有昵称时回退真名，账号菜单沿用现有内容。
- 登录邮箱和用户编号使用原生可理解的折叠信息区，默认不占据视觉中心。
- 编辑资料使用现有桌面弹窗表面，保存完成后刷新当前资料，不跳离主页。

## 手机

- 以 375 × 812 CSS 像素为主要基准，同时检查 320 × 700 与 390 × 844。
- 页面边距 16 像素；封面约 16:7，高度约 150 像素，不能占满首屏。
- 头像约 80 像素并压住封面下缘；身份信息在其下自然展开。
- 本人的“发布动态”和“编辑资料”同排等宽，按钮文字不得逐字换行。
- 账号信息默认折叠；展开后允许复制，但不把邮箱或编号放进公开资料区。
- 编辑资料进入独立全屏任务页，返回主页后保持普通页面层级；不在窄屏中强塞桌面弹窗。
- 内容筛选可横向滚动但应优先完整容纳三个短标签；不使用悬浮发布按钮。

## 编辑与媒体切换

- 用户选择新头像或封面后先在本地预览。普通点击预览不会让焦点瞬间跳到点击位置；只有拖动焦点标记才会连续改变位置。下方水平、垂直滑杆保留为单轴精调和键盘操作入口。
- 头像列在桌面固定为 250 像素，标题、说明、选择操作和正方形预览使用同一宽度；封面操作与`16:7`预览右边缘对齐。图标与“选择图片 / 更换”始终同排，不使用悬浮放大。
- 已经存在封面时不显示“没有封面时的主题”；移除封面后原地展开主题选择，并保留此前选择。
- 弹窗打开后先聚焦当前可见的关闭按钮，不把焦点和滚动位置送到首屏之外的姓名输入框。登录邮箱作为只读、可复制字段显示在个人信息区，与可编辑联系邮箱明确分开。
- 保存时先上传并等待媒体完成隐私处理；只有 `ready` 资源可以成为正式资料媒体。
- 新媒体准备完成后，在一个数据库事务中更新资料文字、主题和媒体关联。旧媒体在关联切换成功且确认无其他引用后再清理。
- 个人信息、可见范围和指定朋友也在同一资料保存事务中更新。展开“指定朋友”后，名单进入当前可视区域但不抢走键盘焦点；没有选择任何朋友时提示保存结果将是私密，并由客户端与服务端共同归一。解除好友后即使旧选择行尚未被清理，也不能继续获得读取权。
- 上传、处理或保存失败时编辑器保持打开，文字、已选文件和焦点不丢失；用户可以继续修改或重试。
- 移除头像时回退到姓名首字或旧 `user.image`；移除封面时立即使用所选主题。
- 资料媒体一旦正式关联即写入 `content_committed_at`。日后移除关联也不会恢复“上传者临时直通”，只能在仍有合法资料关系时读取。
- 桌面底栏以保存为唯一主操作；取消更轻、更窄。两者只用短促的背景、边框和按压反馈，不上浮或放大。

## 状态

- **加载**：资料区使用固定封面、头像和文字骨架，避免内容到达时整体跳动；动态区使用现有骨架节奏。
- **空内容**：本人看到“从一条近况开始”；朋友看到“这里还没有留下可见的片段”；同圈非好友看到“共同经历还没有出现在这里”。空状态说明下一步，不制造虚构活动。
- **筛选为空**：说明当前栏目为空，并提供返回“全部”的明确入口。
- **继续加载**：按钮原位显示短状态，已加载内容保持可读；失败后允许重试。
- **资料保存失败**：在编辑器内说明原因，不关闭编辑器，不清空已写文字或已选图片。
- **权限变化**：关系在页面打开后失效时，后续资料或媒体请求返回 404；界面回到安全的不存在状态，不泄露曾经可见的数据。

## 可复用组件

- `AppShell`：全局导航和手机任务路由层级；
- `UserAvatar`：页眉、朋友、圈子、动态、查看器和个人页共用头像回退；
- `PostStream`：个人页继续使用全站动态展示、编辑、删除和图片查看器；
- `ModalSurface`：桌面资料编辑；
- `ComposerLauncher`：本人发布动态；
- `SegmentedControl`：不用于内容页标签；内容栏目采用可链接、可复用的轻量页签，保证 URL 与浏览器历史可恢复。
