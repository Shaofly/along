# 版本记录

本文件从 `0.3.5` 开始记录每次正式提交的实际变化。早期版本不再逐项追补，避免根据记忆编写不准确的历史。

版本路线与版本记录分工如下：

- [项目 README](../README.md) 的阶段路线说明各阶段计划解决什么问题。
- 本文件说明每个已经发布的小版本实际完成了什么。
- 产品规则、交互规范和技术实现仍以各自专题文档为准。

## 记录规则

- 只在准备正式提交时新增版本，不提前为尚未完成的功能占位。
- 版本号必须与 `package.json`、README 当前状态和提交消息一致。
- “主要变化”只记录用户可感知能力、重要数据模型变化和关键技术调整，不罗列零散 CSS 数值。
- 发布前应完成 lint、TypeScript、文档链接和生产构建检查；存在非阻塞警告时需在交付说明中明确记录。
- Git 提交与推送由项目所有者执行，开发协作者负责更新文档并提供准确提交消息。
