## 状态标记

功能文档必须明确区分产品决定与当前代码，统一使用以下标记：

| 标记 | 含义 |
| --- | --- |
| `已实现` | 当前 Schema、迁移、服务端权限与可用界面已经共同支持；文档应给出主要实现入口 |
| `部分实现` | 核心链路可以使用，但文档中明确列出的某些入口、状态或配套能力尚未完成 |
| `已确认设计` | 产品规则已经确定，后续实现必须遵守，但当前不能作为可用功能宣传 |
| `未来设想` | 仍在探索，可能调整，不构成当前版本承诺 |

每份专题文档开头优先维护“实施状态”表。正文中的设计描述如果尚未实现，必须带状态小节或明确措辞，不能只在文末放一段容易过期的“当前实现”。版本实际交付内容仍以 [版本记录](changelog.md) 和当前代码为准。
