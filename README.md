# 圆个圈（Along）

一个只允许熟人进入、以真实关系和共同回忆为核心的私人社交平台。

相对于微信、QQ 等即时通讯工具，平台更专注于朋友之间的交流与共同回忆，而不刻意追求公开传播、粉丝数量或内容热度。

## 当前状态

当前版本为 `0.3.11`：账号、共同邀请、朋友关系、个人动态、多草稿箱、小圈子第一阶段和安全媒体管线已经可用；本版本增加了多草稿列表、圈子草稿参与者、响应式完整发布器和移动端独立编辑页面。

- Next.js 16、React 19、TypeScript、PostgreSQL 17、Drizzle ORM、Better Auth。
- 支持标准 Node.js standalone 自托管，目标是部署到自租服务器并绑定固定域名。
- 数据库结构由可追踪 SQL 迁移维护；本地媒体经过 Sharp 自动转正、重新编码和隐私元数据清理，持久化生成动态缩略图、查看器预览图和安全高清图。
- 手机、平板和桌面端保持温暖、柔和、低分界感的统一视觉风格。
- 一级页面使用统一响应式页眉；桌面页眉、头像菜单与手机个人抽屉提供草稿箱入口，通知中心只显示不计未读的草稿数量汇总。
- 首页朋友与圈子采用统一摘要组件和真实加载骨架；圈子未读按每次成员周期的服务端阅读位置计算，重新加入不会覆盖旧周期。
- 小圈子使用持久成员关系和独立成员周期记录多次加入与退出；退出后生成个人只读冻结档案，重新加入后恢复查看当前圈子内容并移除旧档案入口。
- 圈子动态与圈子草稿都可显式记录参与者；个人主页只聚合用户当前活跃参与的共同内容，退出后的共同历史从冻结档案进入。
- 同一用户和发布目标可以保存多条服务端草稿；桌面端从草稿列表弹窗继续编辑，手机端进入独立页面。退出圈子或圈子冻结后仍可整理私有草稿，但发布按钮会按最新权限禁用。
- 个人主页与圈子页使用同一套完整发布器：桌面端弹窗、手机端独立任务页；手机发布、草稿编辑和动态编辑统一使用单一返回栏、未保存保护及从右进入/向右退出的空间动效。首页保留轻量快捷发布器且不提供圈子参与者选择，动态编辑暂不支持增删已发布照片。
- 桌面页面、弹窗、原地展开区和主要按钮使用统一 motion tokens；弹窗具备焦点循环、后台隔离与关闭后焦点恢复，破坏性操作使用站内确认而非浏览器原生确认框。
- 发布器重复选择照片时会继续追加并去重，发布按钮显示真实上传百分比和服务端处理状态；动态照片可在桌面端弹层或移动端全屏查看、切换、缩放、拖动和下载，并使用统一的方向判断、退出回弹和缩略图归位逻辑。照片从缩略图按真实几何自然展开和收回，关闭后立即归还页面交互，并修复特定宽高比下相邻轨道边缘泄露的问题。
- 桌面首页将最近动态与圈子、朋友等摘要重新组织为稳定内容栏，使用克制的暖色层次区分区域；照片查看器以独立视口管理四边留白和裁切，只在退出拖动期间允许当前照片越过舞台。
- 带图动态可先进入“发布中”，只有全部图片安全处理完成后才对其他用户可见；临时原文件不参与展示，并由维护任务在 24 小时内回收。

## 快速启动

需要 Node.js 22.13.0 或更高版本、pnpm 和 PostgreSQL 17。

```bash
cp .env.example .env.local
pnpm install
pnpm run db:migrate
pnpm run dev
```

首次打开 `/setup`，使用 `BOOTSTRAP_KEY` 创建管理员和第二位创始成员。完整的 PostgreSQL 初始化、手机测试、Cloudflare Quick Tunnel 和生产构建说明见 [技术架构与部署](docs/architecture.md)。

常用检查：

```bash
pnpm run test
pnpm run build
pnpm run test:smoke
```

## 开发约定

- 产品、权限或交互规则先更新对应文档，再进入开发。
- Schema 变更必须生成新迁移，禁止修改已经应用的历史迁移。
- 所有内容权限必须由服务端校验，不能依赖前端隐藏。
- `.env.local`、数据库数据、上传文件和构建产物不得提交 Git。
- 后续页面继续遵循 [UI 与交互规范](docs/ux.md)，避免版本之间出现割裂的视觉语言。
- 准备版本提交前先核对 Git 历史，并同步 `package.json`、README 当前状态和相关专题文档；开发协作者只整理准确的提交消息，由项目所有者执行提交与推送。
