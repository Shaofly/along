# 安全、隐私与数据处理

本文件记录系统可以诚实承诺的安全边界。尚未实现的能力必须明确写为计划，不能提前作为产品承诺。

## 实施状态

| 能力 | 状态 | 当前边界 |
| --- | --- | --- |
| 登录、会话和受保护接口 | `已实现` | Better Auth、数据库会话与服务端登录校验 |
| 动态、圈子、草稿和媒体读取权限 | `已实现` | Route Handler 与领域查询按当前关系校验 |
| 退出冻结档案权限 | `已实现` | 仅档案所属的已退出成员读取；重新加入后删除旧档案 |
| 私有本地媒体与隐私元数据清理 | `已实现` | 私有目录、Sharp 转正与重新编码、三种正式变体 |
| 上海 COS、数据万象和短期签名地址 | `已确认设计` | 正式云资源尚未接入 |
| 紧急访问、账号注销与数据导出 | `未来设想` | 当前无可用流程，不能提前承诺 |

## 认证与密码

- Better Auth 当前默认使用 `scrypt` 对密码进行不可逆处理，数据库不保存原始密码，管理员不能查看或还原用户密码。
- 暂不为更换算法而自行改造认证核心；后续如切换 Argon2id，需要设计旧密码哈希的兼容迁移。
- 平台不提供公开注册入口；普通成员必须通过绑定邮箱的一次性邀请码注册。
- 会话使用服务端数据库记录和安全 Cookie，所有受保护接口都重新确认登录状态。

## 内容与媒体

- 动态与评论正文不做端到端加密，以支持服务端权限过滤、搜索、共同编辑和回忆整理。
- 应用运行账号必须具备完成业务查询所需的读取能力，不能承诺“数据库账号完全无法读取正文”。
- 管理员后台不提供日常浏览私人正文的入口，管理员身份也不能绕过普通内容权限。
- 当前新图片保存在 `.data/media` 私有目录，通过服务端权限接口读取；正式部署计划迁移到上海私有 COS，并在权限校验后签发短期读取地址。
- 上传图片会自动转正并重新编码，不复制 EXIF、GPS、XMP、IPTC、设备型号和拍摄软件等输入元数据；地图位置只使用用户主动确认的数据。
- 未清理的 incoming 原文件始终私有，不允许被读取接口返回；处理成功后立即删除，异常残留最多保留 24 小时。
- 动态缩略图、查看器预览图和安全高清图是三个独立对象。信息流只读取缩略图，查看器默认读取预览图，高清图只在用户主动请求时读取。
- 退出档案只保存清理后逻辑媒体的引用，不复制未清理原文件。只要档案仍引用媒体，底层文件不能因原动态变化而被回收。

## 服务端权限边界

| 场景 | 默认可见范围 | 可编辑与删除者 | 管理方式是否可切换 |
| --- | --- | --- | --- |
| 个人动态 | 朋友 | 发布者 | 不适用，始终仅自己管理 |
| 指定朋友动态 | 被指定的朋友 | 发布者 | 不适用，始终仅自己管理 |
| 仅我管理的小圈子动态 | 该圈子活跃成员 | 创建者 | 是，仅创建者可升级为共同管理 |
| 共同管理的小圈子动态 | 该圈子活跃成员 | 圈内活跃成员 | 否 |
| 已退出成员的圈子档案 | 退出时冻结且当时有权访问的内容 | 无，只读；本人只能删除整份档案 | 不适用 |
| 个人时间胶囊 | 创建者，开启前不可读 | 封存前的创建者 | 不适用 |
| 共同时间胶囊 | 指定成员 | 封存前的参与成员 | 不适用 |
| 解散圈子的历史内容 | 原圈子成员 | 无，只读 | 不适用 |

所有后端接口都必须在服务器端验证权限。前端隐藏按钮、客户端筛选和管理员身份都不能代替权限校验。地图接口必须先按内容权限筛选，再根据展示级别裁剪坐标。

退出档案不是对当前圈子内容的持续授权。退出后读取的是退出时冻结版本；当前圈子后续修改和新增内容不会进入旧档案。重新加入后旧档案删除，用户重新按持久历史边界读取当前内容。

## 数据库与运维

- 正式环境必须使用 HTTPS，并为数据库、磁盘、备份和对象存储启用静态加密。
- 数据库迁移账号、应用账号、备份账号和人工运维账号应分离，并遵循最小权限原则。
- 日志不得记录密码、完整正文、长期媒体链接或不必要的精确位置。
- 普通管理员不拥有私人内容浏览入口。
- 紧急内容访问尚未实现；未来只有在独立审批、限时授权和可审计记录均完成后才能启用。
- 第一版暂不提供账号注销和数据导出，相关缺失能力统一通过反馈入口说明。

数据库结构约束、迁移账号和备份规则见 [数据库规范](database.md)，图片处理与对象存储边界见 [媒体上传与图片规格](features/media.md)。

## Cloudflare 临时测试

- Better Auth 仅额外信任 `https://*.trycloudflare.com`，Next.js 开发资源仅允许 `*.trycloudflare.com`。
- 该规则用于 Quick Tunnel 测试，不等于信任任意 Cloudflare 托管网站。
- 正式部署必须使用固定域名，并显式配置 `BETTER_AUTH_URL` 与 `TRUSTED_ORIGINS`。

## 后续隐私能力

第一版不单独建设隐私工单或自助账号注销系统。暂未提供的数据导出、删除或隐私能力通过“反馈”入口说明；后续业务成熟后再增加正式的查阅、导出、删除、注销和处理进度流程。
